03 / the mechanism
// how it works under the hood

Not a demo. Not a toy. A training ground.

01

RESET

Agent calls POST /reset. A clean broken repo is copied to /tmp/workspace. Pytest is failing. The clock starts.

02

STEP

Agent reads pytest output. Edits files. Runs commands. Each action costs a step. Each fixed test earns fractional reward. score = pytest_passed / pytest_total

03

GRADE

POST /grade returns the final score: 0.000 to 1.000. Deterministic. Reproducible. Same seed = same bug = same grading. No LLM judge. Just pytest.

04 / task corpus
// three bugs. three lessons. two more for the brave.

Broken by design. Fixed by agents.

[EASY] Syntax error · calculator.py
// A missing colon. The most honest bug.
pytest fails on collection. SyntaxError. Target: read the traceback, fix the line, confirm.
Expected agent score: ~0.85
[MEDIUM] Logic error · data_processor.py
// Code runs. Tests disagree.
No crash. Just wrong answers. Off-by-one. Agent must read assertions, trace the logic.
Expected agent score: ~0.60
[MEDIUM-HARD] Decorator & generator · pipeline.py
// Looks correct. Isn't.
Closures, generators, mutability. Each bug is a subtle Python pattern that reads right on first pass.
Expected agent score: ~0.50
[HARD] Import mismatch · auth.py ↔ test_auth.py
// The traceback lies. The real bug is upstream.
Refactor renamed a function. Caller wasn't updated. Agent must read both files, decide which to fix.
Expected agent score: ~0.35
[VERY-HARD] ABC hierarchy · storage.py
// Four bugs across an inheritance chain.
MemoryStorage, FileStorage, CachingStorage — each has a different failure. copy() only breaks when isolation works.
Expected agent score: ~0.30
